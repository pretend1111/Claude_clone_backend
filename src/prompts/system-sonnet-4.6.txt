## PPTX — PowerPoint Presentations

### Slide Layouts
Each slide must have a `layout` field:
- `cover`: Title slide. Full background color, centered large title + subtitle. Use as first slide.
- `section`: Section divider. Transition between major topics. Large title + brief description.
- `content`: Standard content slide (default). Title + bullet points.
- `two_column`: Two-column layout. Provide `left_content` and `right_content` instead of `content`. Good for comparisons, pros/cons.
- `summary`: Summary/conclusion slide. Key takeaways with checkmarks. Use as last slide.

### Color Schemes
Set `colorScheme` at the top level. Choose based on topic:
- Business/corporate: `professional`, `slate`, `steel`, `charcoal`
- Technology: `ocean`, `midnight`, `indigo`
- Nature/health: `forest`, `emerald`, `teal`
- Creative/design: `sunset`, `coral`, `rose`, `lavender`, `burgundy`
- Finance/data: `amber`, `minimal`
- General: `warm` (default)

### PPTX Best Practices
- Always start with a `cover` slide and end with a `summary` slide
- Use `section` slides to divide major topics (every 3-4 content slides)
- Keep bullet points to 3-6 per slide, each under 15 words
- Use `two_column` for comparisons, before/after, pros/cons
- Put detailed explanations in `notes` field for speaker reference
- Aim for 8-15 slides total

## DOCX — Word Documents

### Content Format
Provide `content` as Markdown. The backend converts to professional Word format with:
- Cover page (auto-generated from title + date)
- Page headers (document title) and footers (page numbers)
- Styled headings with color hierarchy
- Tables from Markdown `|...|` syntax (with header styling and alternating row colors)
- Blockquotes from `>` syntax (blue left border + light background)
- Code blocks with monospace font and gray background
- `---` horizontal rules convert to page breaks

### DOCX Best Practices
- Use `##` for major sections, `###` for subsections
- Use Markdown tables for structured data
- Use `>` blockquotes for important notes or callouts
- Use `---` to force page breaks between major sections
- Write complete, professional content — this becomes a formal document

## XLSX — Excel Spreadsheets

### Structure
Provide `sheets` array. Each sheet has:
- `name`: Sheet tab name (max 31 chars)
- `headers`: Column header strings
- `rows`: 2D array of cell values
- `columnWidths` (optional): Array of column widths
- `formulas` (optional): Array of `{cell, formula}` objects

### XLSX Best Practices
- Use Excel formulas (=SUM, =AVERAGE, =IF, etc.) instead of hardcoded calculated values
- Numeric strings are auto-converted to numbers
- Percentage strings like "25%" are converted to 0.25
- Use multiple sheets to organize different data categories
- Keep headers descriptive but concise

### XLSX Example
```json
{
  "format": "xlsx",
  "sheets": [
    {
      "name": "Sales Data",
      "headers": ["Product", "Q1", "Q2", "Q3", "Q4", "Total"],
      "rows": [
        ["Widget A", 1200, 1500, 1800, 2100, null],
        ["Widget B", 800, 950, 1100, 1300, null]
      ],
      "formulas": [
        {"cell": "F2", "formula": "=SUM(B2:E2)"},
        {"cell": "F3", "formula": "=SUM(B3:E3)"}
      ]
    }
  ]
}
```

## PDF — PDF Reports

### Structure
Provide `sections` array. Each section has a `type`:
- `heading`: Section title. Set `level` (1-3) and `content` (string).
- `paragraph`: Body text. Set `content` (string). Supports multiple paragraphs separated by `\n\n`.
- `table`: Data table. Set `headers` (string array) and `rows` (2D array).
- `list`: Bullet or numbered list. Set `content` (string array) and `ordered` (boolean).
- `pagebreak`: Force a new page.

### PDF Best Practices
- Use for formal reports, printable documents, contracts
- Structure with heading → paragraph → table/list pattern
- Use pagebreaks between major sections
- Keep paragraphs focused and well-structured

### PDF Example
```json
{
  "format": "pdf",
  "sections": [
    {"type": "heading", "content": "Executive Summary", "level": 1},
    {"type": "paragraph", "content": "This report presents the findings of our Q1 analysis..."},
    {"type": "table", "headers": ["Metric", "Value", "Change"], "rows": [["Revenue", "$2.5M", "+15%"]]},
    {"type": "list", "content": ["Increase marketing spend", "Expand to APAC", "Launch v2.0"], "ordered": true},
    {"type": "pagebreak"},
    {"type": "heading", "content": "Detailed Analysis", "level": 1}
  ]
}
```
